# ImageSuite-Wan2GP — Audit Report

<sub>Generated by a 23-agent multi-agent audit (2026-06-15) — 57 feature ideas, 12/12 candidate bugs verified against source.</sub>

A capable image workbench with a striking amount of completed-but-unwired backend. The single biggest theme: **~600 lines of ControlNet, an SDXL refiner, IP-Adapter FaceID/body-double, BiRefNet segmentation, custom-VAE loading, and Laplacian blending are all implemented and dead** — activating them is the highest leverage available. Alongside that, a handful of genuine bugs (one browser-hang, one touch-device data-loss, one "send" that silently does nothing, and a sampler-config leak that quietly corrupts output) deserve fixing first.

## 1. TL;DR — build these next (ranked)

1. **Wire ControlNet end-to-end** (pose/canny/depth/lineart/tile) into Txt2Img/Img2Img/Inpaint — the entire `controlnet_types.py` registry + `generate_controlnet` (`sd_pipeline.py:1434`) is dead code; the biggest unrealized feature. (L, high)
2. **Embed A1111 PNG metadata + a "read params" recall** — no provenance is saved today (zero `PngInfo` in the tree); makes every output reproducible/shareable. (M, high)
3. **Surface the actual seed used + Reuse-seed button** — `Seed=-1` rolls a random seed that is never returned, even though it's literally in the `sd_<seed>_*.png` filename. (S, high)
4. **One-click subject auto-mask via BiRefNet** (`segmentation.segment_foreground`, fully built/cached, unwired) into MultiCanvas mask + a Modify "Remove background." (M, high)
5. **Magic-wand tolerance slider + add/subtract modifiers** — tolerance is hardcoded at 40; the #1 control users reach for is invisible. (S, high)
6. **TAESD live latent preview during SD sampling** — currently zero visual feedback for 10-30s; ~10MB VRAM, lets users abort bad seeds early. (M, high)
7. **Distilled turbo presets (LCM/Hyper-SD/Lightning)** — 4-8-step SDXL is the iterate-loop win on a 12GB card; all plumbing (LoRA loader, LCMScheduler) exists. (M, high)
8. **Wire the SDXL refiner + Latent hi-res fix** — `_run_refiner`/`_load_refiner` accept `refiner_*` args nothing passes; hi-res fix reuses existing img2img. (S/M, high/medium)
9. **Rotate/Straighten + Undo stack in Modify** — editor can only flip; Apply-crop is irreversible (and Reset-crop is broken, see bugs). (M, high)
10. **Permanent organized output library** — SD gens write only to `.cache/sd_gen` that Flush wipes; `outputs_dir()` exists but nothing routes to it. (M, high)

## 3. Feature & improvement catalogue

### Generation & models
- **Wire ControlNet (pose/canny/depth/lineart/scribble/tile)** into Txt2Img/Img2Img/Inpaint — full registry + `generate_controlnet` + MultiControlNet exist, called from nowhere. UI: 1-3 slots, type dropdown, preprocess preview, strength/start/end. SDXL-gated; single CN fits 12GB via existing sequential offload. **(L, high)** — *activate existing backend.* `ui/page.py`, `plugin.py`, `core/gen_sd.py` (thin `run_controlnet`), `core/sd/sd_pipeline.py:1434`, `core/sd/controlnet_types.py`.
- **Wire the SDXL refiner** — `generate_txt2img` accepts `refiner_checkpoint/refiner_switch_at/refiner_steps/refiner_cfg_scale`; nothing passes them, no UI. Add an SDXL-only accordion; give `_load_refiner` the `mem_get_info` offload guard used in `_get_inpaint`. **(S/M, high)** — *activate existing backend.* `core/gen_sd.py`, `ui/page.py`, `core/sd/sd_pipeline.py:244`.
- **Latent hi-res fix** — base gen → upscale 1.5-2x → low-denoise img2img second pass; avoids SDXL subject-duplication at high res. Reuses `_get_img2img_pipe`. **(M, high)** net-new orchestration.
- **Distilled turbo presets (LCM/Hyper-SD/Lightning/DMD2)** — profile table → inject distill LoRA + force sampler/steps/CFG. 3-6x faster, ideal for 12GB iterate loop. **(M, high)** `core/gen_sd.py`, `ui/page.py`.
- **compel prompt weighting / BREAK / >77-token chunking** — prompts go in as raw strings today, so `(word:1.3)` weights are ignored and >77 tokens silently truncated — exactly what Pony/Illustrious users rely on. **(L, high)** `core/deps.py` (auto-install), `core/sd/sd_pipeline.py` (dual-encoder Compel), `core/gen_sd.py`.
- **Fix Sampler/Scheduler dropdowns to match backend** — UI offers "DPM++ 2M SDE", "Heun", "SGM Uniform", "Simple", "default", "Normal" that aren't in `_SAMPLER_MAP`/`_SCHEDULER_OVERRIDES`, so `create_scheduler` silently falls back to Euler/Automatic. Source choices from `list_samplers()/list_schedulers()`; add SGM-Uniform via `timestep_spacing='trailing'`. **(S, medium)** `ui/page.py:25-27`, `core/sd/sd_samplers.py`. *(Note: "DPM++ 2M SDE" actually IS in the map at `sd_samplers.py:21` — verify which value the UI sends.)*
- **FreeU + CFG-rescale (+ later PAG)** — cheap inference-only quality knobs; rescale fights the high-CFG "fried" look. No extra VRAM. **(M, medium)** `core/sd/sd_pipeline.py`, `ui/page.py`.
- **Wire custom VAE selection** — `_load_vae` (`sd_pipeline.py:143`) is built but `sd_vae_dir` is hardcoded `''` (`gen_sd.py:28`) so it always early-returns; can't use `sdxl-vae-fp16-fix`. Add a dir setting + `discover_vaes`. **(S, medium)** — *activate existing backend.*
- **Parameter sweep / XYZ grid** — batch count is only N identical images today; add axis pickers + value lists, cartesian product over the existing per-image path. **(L, medium)** `plugin.py`, `ui/page.py`.

- **Activate FaceID / full-face IP-Adapter variants** — Color Reference hardcodes `ip-adapter-plus_sdxl`; `_extract_face_embeddings` + faceid-plus dual-encoder path are built. True identity transfer. **(M, high)** — *activate existing backend.*

### Performance & VRAM (12GB-aware)
- **TAESD live latent preview** (TL;DR #6) — ~10MB, single-digit-ms decode; free the TAESD in `release_all`. **(M, high)**
- **Prefer PyTorch SDPA over xformers + channels_last UNet** — `load()` hard-relies on `enable_xformers` (`sd_pipeline.py:104`); on torch≥2 SDPA matches/beats it, drops a fragile dep, and channels_last is a free Ampere conv win. **(S, medium)**
- **fp8 / sequential-offload memory-policy toggle + keep-resident option** — `load()` unconditionally `.to('cuda')`; release frees everything so the next gen re-reads 6.5GB from disk. Generalize the proven `mem_get_info` heuristic from `_get_inpaint`. **(M, medium)**

## 4. Phased roadmap

### Phase 0 — fix what's broken (do first; mostly S)
- **Bugs 1, 5, 6** (canvas: flood-fill hang, touchend crash, button filter) — pure JS, protect unsaved work. **S.**
- **Bug 4** (`goto_media_tab` rename) and **Bug 12** (`"out"` exclude) — one-liners. **S.**
- **Bug 2** (sampler config leak) and **Bug 3** (full-res inpaint stretch) — correctness; both have a proven in-repo pattern to copy. **S/M.**
- **Bugs 7, 9, 10, 11, 8** — round out correctness. **S.**

### Phase 1 — quick wins, high impact (S, mostly client-side or near-free)
- Magic-wand **tolerance slider + add/subtract** (S). Brush **shortcuts** (S).
- **Seed surfacing + Reuse-seed** (S). **Auto-interrogate + threshold + append** (S).
- **`release_all` CN/body-double teardown** (S) — *activate-existing; prerequisite for Phase 3.*
- **SDPA-over-xformers + channels_last** (S). **Sampler/scheduler dropdown reconciliation** (S).
- **Laplacian seamless blend toggle** (S) — *activate-existing.* **Custom VAE wiring** (S) — *activate-existing.*

### Phase 2 — mid bets, high value (M)
- **PNG metadata embed + recall** (M, net-new). **Output history browser + organized output library** (M).
- **BiRefNet subject auto-mask (canvas) + Remove-background (Modify)** (M) — *activate-existing.*
- **TAESD live preview** (M). **Turbo presets** (M). **SDXL refiner wiring** (M) — *activate-existing.*
- **Modify rotate/straighten + undo stack** (M). **Before/after compare slider** (M).
- **FaceID IP-Adapter variants** (M) and **Face restore on Modify** (M) — *activate-existing.*
- **Hi-res fix** (M). **Memory-policy toggle** (M).

### Phase 3 — bigger bets (L), 12GB-aware, gated
- **ControlNet end-to-end** (L) — *activate-existing*; ship single-CN first (one CN + SDXL fits 12GB via sequential offload), gate multi-CN behind a VRAM warning, and land the preprocessor-hardening probe alongside. **Depends on Phase 1 `release_all` teardown.**
- **Replace Person / Body Double** (L) — *activate-existing*; builds on FaceID + ControlNet.
- **compel prompt weighting** (L, net-new). **AI upscale (tile-CN/img2img refine)** (L) — fill the empty tile `sdxl_repo`. **XYZ sweep grid** (L). **Folder/batch processing** (L) — wire `swap_frame`.

**12GB notes:** every Phase 3 GPU feature relies on the pipeline's existing sequential/model CPU-offload (proven in `_get_inpaint`, `_get_controlnet_pipe`, `_get_body_double_pipe`). The one hard prerequisite is freeing CN/body-double pipes in `release_all` before handing the GPU back to Wan2GP video gen — without it, Phase 3 will OOM the card.
